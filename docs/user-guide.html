MITx-Canvas LTI Link Updater — User Guide

What this tool does

MITx links in your Canvas course point to a specific version of your MITx course. When a new version is created for a new semester, those links still point to the old one. This tool updates every MITx link to the new version and gives you an updated Canvas file to import, plus a report of what changed. It takes about 10 minutes, once a year.

Getting your two export files

Old Canvas course (the one with the MITx links):

  1. In Canvas, go to Settings → Export Course Content
  2. Select "Course" and click "Create Export"
  3. Download the file (ends in .imscc)

New MITx course:

  1. In MITx Studio, open the new semester's course
  2. Go to Tools → Export and click "Export Course Content"
  3. Download the file (ends in .tar.gz)

You don't need to rename or open either file — the tool identifies them automatically.

Using the tool

  1. Drag both files into the box on the main page, in any order, and click "Check Files"
  2. Review the confirmation screen — it shows which course your links point to now and where they'll point after the update. If anything looks unexpected, cancel and re-export the correct course. Nothing changes until you confirm.
  3. Confirm. In Chrome or Edge you'll choose where to save the updated file; other browsers download it automatically. Download the report too if you want it.

Importing into Canvas

  1. In Canvas, open the new semester's course
  2. Go to Settings → Import Course Content
  3. Select "Canvas Course Export Package" and upload the _updated.imscc file
  4. After importing, click a few MITx links in Student View to confirm they open the new course's content

Understanding the report

StatusMeaningAction
MATCHEDLink found in the new MITx course and updatedNone
MISSINGLink points to content not found in the new course; left unchangedUpdate or remove the link in Canvas manually
NEW_ONLYThe new course has content with no Canvas linkOptional — add a link if students should reach it

The MITx tool configuration link (just https://lms.mitx.mit.edu) always shows as MISSING. That's normal — it isn't a content link.

Advanced: no MITx export? (unverified update)

If you can't get the new MITx course export but you know the new course is an exact copy of the old one (a re-run or copy made by the MITx team — the usual case for a new semester), you can use the advanced option on the main page:

  1. Expand "Advanced: don't have the new MITx export?" and tick the checkbox
  2. Enter the new course's ID, e.g. course-v1:MITx+course_name+3T2026 — your MITx course team contact can give you this
  3. Drop in just the old Canvas export (.imscc) and continue as normal

Because the tool can't check the links against the new course in this mode, the report is marked UNVERIFIED REWRITE. After importing, click two or three MITx links in Student View to confirm they work. If any don't, contact mitx-support@mit.edu.

Troubleshooting

  • "No MITx links found" — you probably exported the new Canvas course instead of the old one; the MITx links live in last semester's course.
  • Many links show as MISSING — usually the wrong MITx course was exported. Double-check the new course export and try again.
  • It warns the files look like the same course version — one export came from the wrong place; re-export the old Canvas course and the new MITx course.
  • Links still point to the old course after importing — the original file was imported instead of the one ending in _updated.imscc; re-import the updated file.

For anything else, email mitx-support@mit.edu with a short description and, if you have it, the report from the tool.
